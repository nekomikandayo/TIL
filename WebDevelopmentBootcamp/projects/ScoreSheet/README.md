# 卓球得点管理アプリ (Ping Pong Score Keeper)

## 概要
JavaScriptのDOM操作とイベント処理の基礎を定着させるために作成した演習プロジェクトです。

## 学習の狙い
* **DOMイベントの実践:** `click` や `change` イベントを使って、ユーザーの操作に合わせて画面を書き換える処理を学ぶ。
* **データと見た目（UI）の分離:** 画面の文字（DOM）を直接計算に使うのではなく、JS内の変数（オブジェクト）で点数を管理し、それを画面に反映させるという設計を意識しました。

## 開発時の工夫と学び
* **AIを「メンター」として活用:**
  ゼロからの設計で手が止まった際、AIにコードの答えを聞くのではなく「デバッグのヒント」や「設計の考え方」だけを引き出すように質問を重ねました。これにより、ブラックボックスを作らずに本質的なロジックを理解しながら実装できました。
* **DOM取得の効率化:**
  ボタンを押すたびに `querySelector` で要素を探すと無駄な処理が発生するため、ファイルの上部で一度だけ変数に取得（キャッシュ化）し、それを関数内で使い回す仕組みにしました。
* **型の違いによるバグ防止:**
  `<select>` から取得した値が「文字列」になる仕様に躓きましたが、`parseInt()` で明示的に数値へ変換してから比較することで、予期せぬ計算バグを防ぎました。
* **リセット処理の徹底:**
  点数を0に戻すだけでなく、勝敗確定時に付与したCSSクラス（文字色）も確実に `classList.remove` で消す処理をリセット関数にまとめ、2ゲーム目以降の表示バグをなくしました。
